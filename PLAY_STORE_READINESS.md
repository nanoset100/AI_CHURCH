## 📋 출시까지 권장 순서

### Phase 1: Android 앱으로 빌드 가능하게 (필수)

1. **Android 플랫폼 추가**  
   프로젝트 루트에서:
   ```bash
   flutter create . --platforms=android
   ```
2. **릴리스 빌드 확인**
   ```bash
   flutter build appbundle
   ```
3. **패키지명·버전**  
   `android/app/build.gradle` 에서 `applicationId`, `versionCode`/`versionName` 확인·수정

### Phase 2: 스토어 필수 자료 준비

1. **개인정보처리방침**  
   - Supabase(이메일, 프로필 등) 수집 항목 명시  
   - 호스팅할 URL 준비 (웹사이트, GitHub Pages 등)
2. **스토어 문구**  
   - 앱 이름, 짧은 설명(80자), 자세한 설명(4000자 이내)
3. **스크린샷**  
   - 휴대폰 필수 2장 이상 (권장 4~8장)
4. **앱 아이콘**  
   - 512x512 PNG (투명 배경 불가)
5. **기능 그래픽**  
   - 1024x500 이미지 1장

### Phase 3: 앱 품질 (선택이지만 권장)

1. **인사말**  
   - Supabase `profiles.display_name`(또는 이메일) 연동
2. **오늘의 예배/성경**  
   - Supabase 등 백엔드 데이터 연동 또는 “준비 중” 안내
3. **네트워크 오류**  
   - 인터넷 끊김·실패 시 스낵바/다이얼로그 메시지
4. **크래시 방지**  
   - Firebase Crashlytics 등 (선택)

## 한 줄 요약

- **지금 상태:**  
  웹/기능 측면에서는 “동작하는 MVP” 수준이고,  
  **Play 스토어에 그대로 제출할 수 있을 정도로는 아직 완전하지 않습니다.**

- **출시하려면:**  
  1) Android 프로젝트 추가 후 `appbundle` 빌드 가능하게 하고,  
  2) 개인정보처리방침·데이터 보안·스토어 등록정보·스크린샷·아이콘 등 **필수 항목**을 채운 뒤,  
  3) 가능하면 인사말·오늘의 예배/성경 등 **실제 데이터 연동**과 **오류 처리**를 보강하는 것을 권장합니다.
